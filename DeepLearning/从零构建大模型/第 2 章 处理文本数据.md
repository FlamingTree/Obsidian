### 一、先搞懂 2 个核心基础概念
这段内容的核心是大语言模型的底层处理逻辑，先把最基础的概念讲明白，后续内容就会非常好理解：
1.  **词元（Token）**：大语言模型没法直接读懂人类的文字，只能处理数字。所以我们会用「分词器」把文本拆分成一个个最小的处理单元，这个单元就叫词元（Token），之后每个词元会被映射成对应的数字，再喂给模型。
    举个例子：“今天天气很好”，分词器可能会拆成 [" 今天 ", " 天气 ", " 很 ", " 好 "] 这 4 个词元，再转成模型能识别的数字 ID。
2.  **特殊词元**：这类词元不对应自然语言里的文字，作用是给模型传递**控制信号**，比如告诉模型“文本到此结束”“这是用户的提问”等，是模型训练和生成的重要指令。

### 二、逐句拆解核心内容
#### 1. GPT 的极简特殊词元设计：<|endoftext|> 与 [EOS]
> 值得注意的是，GPT 模型使用的分词器并不依赖这些特殊词元，而是仅使用<|endoftext|>词元来简化其处理流程。<|endoftext|>词元与 [EOS] 词元作用相似。

这部分讲的是 GPT 对特殊词元的极简设计，核心信息有 2 点：
- 先解释 [EOS]：它是 `End of Sequence` 的缩写，翻译为「序列结束符」，是 NLP（自然语言处理）领域最经典的特殊词元之一。它的核心作用，就是**明确告诉模型：这段文本的有效内容到这里就完全结束了**。
  训练时，模型看到 [EOS] 就知道一篇文档/句子的边界；生成文本时，模型一旦生成 [EOS]，就会停止继续输出。
- 再讲 GPT 的<|endoftext|>：GPT 系列模型（尤其是早期的 GPT1/GPT2/基础 GPT3）没有设计一堆五花八门的特殊控制词元，只靠<|endoftext|>这一个核心特殊词元，就完整承担了 [EOS] 的全部功能。
  预训练时，它用来标记海量语料里一篇文档的结束；生成时，模型输出它就会停止生成。这就是原文说的“作用相似”，同时只用这一个词元，也极大简化了模型的分词和文本处理流程。

#### 2. <|endoftext|>的第二个作用：文本填充（Padding）
> 此外，<|endoftext|>也被用于文本的填充。

这里要先讲清楚：为什么需要“填充”？
大模型训练不是一条一条文本喂进去的，而是**一批一批（Batch）批量输入**的。而模型的输入有一个硬性要求：同一个批次里，所有文本转换成词元后的长度必须完全一致，不然没法拼成统一格式的张量（Tensor），无法进行并行计算。
但现实里，不同文本的长短天差地别，比如有的句子只有 10 个词元，有的有 100 个词元。为了统一长度，就需要「填充（Padding）」：
给短文本的末尾，补上一堆无意义的填充词元，把它的长度拉长到和批次里最长的文本一致，让整个批次的所有样本长度完全统一。

而 GPT 的设计，是直接复用已经存在的<|endoftext|>来做填充词元，不用再额外设计一个专门的 [PAD]（填充符）特殊词元，进一步简化了分词器的设计。

#### 3. 掩码技术：为什么填充的词元完全不重要
> 然而，正如本书后面章节中将要探讨的那样，当模型在批量输入上进行训练时，我们通常使用掩码技术，这意味着我们并不会关注那些仅用于填充的词元。因此，具体选择哪种词元来进行填充实际上并不重要。

这是这段内容的核心结论，我们分两层讲透：
- 第一层：为什么需要掩码技术？
  填充的词元，是我们为了统一长度硬补进去的，**不是文本的真实有效内容**。而 Transformer 模型的核心是自注意力机制——模型处理每个词元时，会去关注句子里其他所有词元的信息。
  如果让模型去关注这些无意义的填充词元，会学到错误的干扰信息，严重影响训练效果。所以我们会用**注意力掩码（Attention Mask）** 技术，给模型明确划定规则：
  哪些位置是真实的文本词元，需要正常计算注意力；哪些位置是填充的词元，直接完全忽略，计算时彻底屏蔽这些位置的信息。

  举个例子：批次里最长的文本有 100 个词元，有一条文本只有 10 个真实词元，我们补了 90 个<|endoftext|>做填充。对应的掩码就会给前 10 个位置标“有效”，后 90 个填充位置标“无效、忽略”。模型在训练计算时，完全不会看、不会学习这 90 个填充位置的任何内容。

- 第二层：最终结论的逻辑
  既然掩码技术已经让模型彻底忽略了所有填充的词元，这些填充内容根本不会进入模型的学习和计算，那**用什么词元来做填充，本质上没有任何区别**。
  你可以用<|endoftext|>，可以专门设计一个 [PAD] 词元，甚至用一个冷门的普通词元，都完全不会影响模型的训练效果。这就是原文最后一句话的核心含义。

这段内容完整讲清了 GPT 分词器的极简设计理念，以及配套的批量训练工程逻辑：
1.  设计极简：GPT 不用复杂的特殊词元体系，仅靠<|endoftext|>一个词元，就承担了「文本结束标记（EOS）」的核心功能，简化了处理流程。
2.  一物多用：直接复用<|endoftext|>作为填充词元，解决批量训练时文本长度不统一的问题，不用额外新增特殊词元。
3.  技术兜底：训练时通过注意力掩码技术，让模型完全忽略所有填充词元，填充内容不会对模型产生任何影响。
4.  最终结论：填充词元的选择无关紧要，不会影响模型的训练和效果。

### 一、先搞懂核心前提：为什么大模型必须注入「位置信息」？
这段内容的开篇，是整个问题的根源，我们先把这句话拆透：
> 原则上，带有确定性且与位置无关的词元 ID 嵌入能够提升其可再现性。然而，由于大语言模型的自注意力机制本质上与位置无关，因此向模型中注入额外的位置信息是有帮助的。

我们分两部分拆解，先补 2 个基础概念：
- **词元 ID 嵌入（Token Embedding）**：大模型无法直接读懂文字，会先把每个词元（Token）转换成一个固定长度的向量，这个向量就是「词元嵌入」。同一个词元（比如“苹果”），不管它在句子的开头还是结尾，它的词元 ID 是固定的，对应的嵌入向量也是固定的，这就是**确定性、与位置无关**。
- **可再现性**：正因为同一个词的语义向量不随位置变化，模型才能稳定识别这个词的核心含义，不会因为它出现在句子的不同位置，就丢失或改变本身的语义，保证了语义识别的稳定性和可复现性。

#### 关键问题：为什么必须额外加位置信息？
核心原因是：**Transformer 的自注意力机制，天生是「排列不变」的——它只关心句子里有哪些词，完全不关心词的先后顺序**。

举个最直观的例子：
- 句子 A：`小明 吃 了 苹果`
- 句子 B：`苹果 吃 了 小明`

这两句话的词元完全一致，只是语序不同，但语义天差地别。但如果没有位置信息，纯自注意力机制会把这两句话当成完全相同的内容——它只会计算词和词之间的关联，完全分不清谁在前、谁在后。

而人类语言里，**语序是语义的核心载体**，没有位置信息，模型根本无法理解句子的真实含义。这就是为什么必须给模型额外注入位置信息，这也是位置嵌入技术的核心意义。

### 二、第一种方案：绝对位置嵌入（Absolute Positional Embedding）
> 绝对位置嵌入直接与序列中的特定位置相关联。对于输入序列的每个位置，该方法都会向对应词元的嵌入向量中添加一个独特的位置嵌入，以明确指示其在序列中的确切位置。例如，序列中的第一个词元会有一个特定的位置嵌入，第二个词元则会有另一个不同的位置嵌入，以此类推。

#### 1. 核心逻辑
它的核心是**给「句子里的第几个位置」绑定一个专属、唯一的向量**。
比如：句子里第 0 个位置（索引从 0 开始），有一个固定的位置向量 `pos_0`；第 1 个位置，有独一无二的 `pos_1`；第 2 个位置是 `pos_2`……以此类推，每个位置的向量只和“位置编号”绑定，和这个位置放什么词元完全无关。

#### 2. 实现方式
非常简单直接：**最终输入给模型的向量 = 词元本身的嵌入向量 + 对应位置的位置嵌入向量**。
两个向量相加后，既保留了词本身的核心语义，又给模型明确传递了「这个词在句子的第几个位置」的信息。

#### 3. 通俗例子
还是以 `小明 吃 了 苹果` 为例，4 个词元对应位置 0、1、2、3：
- “小明”的词嵌入是 `E_小明`，加上位置 0 的向量 `pos_0`，得到最终输入向量；
- “吃”的词嵌入是 `E_吃`，加上位置 1 的向量 `pos_1`；
- “了”的词嵌入是 `E_了`，加上位置 2 的向量 `pos_2`；
- “苹果”的词嵌入是 `E_苹果`，加上位置 3 的向量 `pos_3`。

这样一来，哪怕是同一个词，出现在不同位置，最终的输入向量也完全不同，模型就能精准区分它的位置，解决了自注意力的排列不变问题。

#### 4. 特点与局限
- 优势：实现简单、逻辑直观，早期经典模型（BERT、初代 GPT）都采用了这种方案；
- 短板：泛化能力差。如果训练时模型最多见过长度 512 的句子，那它只学到了 `pos_0` 到 `pos_511` 的向量。推理时如果输入超过 512 长度的句子，超出部分的位置是模型从未见过的，就会处理失效，无法适配训练外的超长序列。

### 三、第二种方案：相对位置嵌入（Relative Positional Embedding）
> 相对位置嵌入关注的是词元之间的相对位置或距离，而非它们的绝对位置。这意味着模型学习的是词元之间的“距离”关系，而不是它们在序列中的“具体位置”。这种方法使得模型能够更好地适应不同长度（包括在训练过程中从未见过的长度）的序列。

#### 1. 核心逻辑
它完全抛弃了“位置编号”的概念，核心是**只关心两个词元之间的「相对距离 + 前后关系」，不关心它们在句子里的绝对位置**。
比如：A 词在第 3 位，B 词在第 5 位，那 A 和 B 的相对关系是「B 在 A 的后面 2 位（相对距离 +2）」；哪怕换到更长的句子里，A 在第 100 位，B 在第 102 位，相对距离还是 +2，模型会用完全一样的规律处理这两种情况。

#### 2. 实现方式
和绝对位置嵌入不同，它不是在输入层直接把位置向量加到词嵌入上，而是**把相对位置信息融入到自注意力的权重计算过程中**。
模型会提前学习：当两个词相隔 k 个位置时，它们的注意力权重应该如何调整。最终模型学到的是「不同相对距离下的注意力规律」，而不是固定位置的向量。

#### 3. 通俗例子
还是以 `小明 吃 了 苹果` 为例：
- 计算“小明”（位置 0）对“苹果”（位置 3）的注意力时，相对距离是 +3，模型用学到的「相隔 3 位」的规律调整权重；
- 换一个更长的句子：`昨天 小明 买 了 水果 他 吃 了 苹果`，“小明”在位置 1，“苹果”在位置 8，相对距离是 +7，模型直接复用学到的「相隔 7 位」的规律即可。

哪怕这个句子的长度，比模型训练时见过的所有句子都长，只要词和词之间的相对距离是可计算的，模型就能稳定处理，不会因为绝对位置超出训练范围而失效。

#### 4. 核心优势
完美解决了绝对位置嵌入的长序列泛化短板，对不同长度的序列适配性极强，尤其是超长文本场景。目前主流的大语言模型（比如 LLaMA 系列、GPT3 之后的迭代模型），大多采用相对位置嵌入或其改进方案（比如现在最主流的 RoPE 旋转位置编码，就属于相对位置嵌入的优化方案）。

### 四、两种方案核心对比总结
| 对比维度      | 绝对位置嵌入               | 相对位置嵌入                        |
| :-------- | :------------------- | :---------------------------- |
| **核心关注点** | 词元在序列里的「绝对位置编号」      | 两个词元之间的「相对距离/前后关系」            |
| **实现方式**  | 输入层直接给词嵌入加对应位置的固定向量  | 融入自注意力计算，学习相对距离的注意力规律         |
| **核心优势**  | 逻辑简单、实现容易，早期经典模型广泛使用 | 长序列泛化能力极强，适配训练未见过的文本长度        |
| **核心短板**  | 无法适配超出训练长度的序列，泛化性差   | 实现逻辑更复杂，计算成本略高于基础绝对位置嵌入       |
| **典型应用**  | 早期 BERT、初代 GPT       | 现代主流大模型（LLaMA、GPT-4 等）、超长文本模型 |

> [!NOTE] <font color="#c00000">位置嵌入向量是如何生成的</font>

位置嵌入向量的生成方式主要分为两大类：**绝对位置编码**（可学习、正弦余弦）和**相对位置编码**（经典相对、RoPE 旋转）。不同的生成方法逻辑差异极大，我会逐一拆解其核心原理、数学公式，并配合通俗例子和代码片段，帮你彻底搞懂。

### 一、绝对位置嵌入的生成方式
绝对位置嵌入的核心是：给序列中的第 `pos` 个位置（从 0 开始计数），生成一个唯一的固定向量。主要有两种主流生成方案。

#### 1. 可学习的位置嵌入（Learned Positional Embedding）
这是最简单直观的方式，早期的 BERT、GPT-2 都采用这种方案。

**生成逻辑**：
1.  **初始化**：直接随机初始化一个巨大的矩阵 `Position_Embedding`，形状为 `[max_seq_len, d_model]`。
    - `max_seq_len`：模型预设能处理的最大序列长度（比如 512、1024、4096）；
    - `d_model`：模型的词嵌入维度（比如 BERT-base 是 768，GPT-2 是 768/1024）。
2.  **训练更新**：这个矩阵会作为模型的可训练参数，在预训练过程中，通过反向传播和梯度下降，和词嵌入矩阵一起被自动学习和优化。
3.  **使用时**：输入序列的第 `pos` 个词，直接从矩阵里取出第 `pos` 行的向量，加到词嵌入上即可。

**Python 代码示例（PyTorch）**：
```python
import torch
import torch.nn as nn

# 假设最大长度 512，嵌入维度 768
max_seq_len = 512
d_model = 768

# 定义可学习的位置嵌入层
learned_pos_emb = nn.Embedding(max_seq_len, d_model)

# 生成位置索引 [0, 1, 2, ..., 9]（假设输入长度为10）
positions = torch.arange(0, 10, dtype=torch.long)

# 取出对应的位置嵌入向量
pos_embedding = learned_pos_emb(positions)
print(pos_embedding.shape)  # 输出: torch.Size([10, 768])
```

**特点**：
- ✅ 优点：实现极其简单，不需要设计复杂的数学公式，完全靠数据驱动学习。
- ❌ 缺点：无法外推。如果推理时输入长度超过了 `max_seq_len`，超出部分的位置没有对应的预训练向量，模型无法处理。

---

#### 2. 正弦余弦位置编码（Sinusoidal Positional Encoding）
这是《Attention Is All You Need》原论文中提出的**固定函数式**位置编码，不需要学习，直接通过数学公式计算生成。

**核心数学公式**：
对于序列中的第 `pos` 个位置，嵌入向量的第 `i` 个维度（偶数位和奇数位分开计算）：

$$
\begin{cases}
PE_{(pos, 2i)} = \sin\left( \frac{pos}{10000^{2i / d_{\text{model}}}} \right) \\
PE_{(pos, 2i+1)} = \cos\left( \frac{pos}{10000^{2i / d_{\text{model}}}} \right)
\end{cases}
$$

**公式通俗解释**：
- `pos`：词元在序列中的绝对位置（0, 1, 2, ...）；
- `i`：嵌入向量的维度索引（0, 1, 2, ..., d_model-1）；
- `d_model`：嵌入向量的总维度；
- **核心思想**：用不同频率的正弦和余弦函数，生成周期性的向量。每个维度对应一个不同的“波长”，从短到长覆盖各种频率，从而编码任意位置的信息。

**Python 代码实现**：
```python
import torch
import math

def get_sinusoidal_pos_encoding(max_len, d_model):
    # 初始化位置编码矩阵
    pe = torch.zeros(max_len, d_model)
    
    # 生成位置索引 [0, 1, ..., max_len-1]，转为列向量
    position = torch.arange(0, max_len, dtype=torch.float).unsqueeze(1)
    
    # 计算分母项：10000^(2i/d_model)
    div_term = torch.exp(torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model))
    
    # 偶数维度用 sin，奇数维度用 cos
    pe[:, 0::2] = torch.sin(position * div_term)
    pe[:, 1::2] = torch.cos(position * div_term)
    
    return pe

# 示例：生成最大长度 10，维度 4 的位置编码（方便观察）
sinusoidal_pe = get_sinusoidal_pos_encoding(max_len=10, d_model=4)
print("位置 0 的编码:", sinusoidal_pe[0])
print("位置 1 的编码:", sinusoidal_pe[1])
```

**特点**：
- ✅ 优点 1：**可以外推**。哪怕推理时的序列长度比训练时长得多，也可以直接用公式计算出新位置的编码，不需要重新训练。
- ✅ 优点 2：有明确的数学规律性，理论上模型可以更容易地学习到相对位置关系（比如 `pos+k` 的编码可以通过 `pos` 的编码线性变换得到）。
- ❌ 缺点：相比可学习的方式，它是人为设计的固定函数，在数据量极大时，拟合能力略逊于可学习嵌入。

### 二、相对位置嵌入的生成方式
相对位置嵌入不关注“绝对位置编号”，而是关注**两个词元之间的相对距离**。这里讲两种最主流的方案：Transformer-XL 的经典相对位置编码，以及现在大模型（LLaMA、GPT-NeoX 等）通用的 **RoPE（旋转位置编码）**。

#### 1. RoPE：旋转位置编码（Rotary Positional Embedding）—— 目前最主流
RoPE 是目前开源大模型（LLaMA 1/2/3、Qwen、Baichuan 等）的标配，它通过**旋转矩阵**的方式，巧妙地将相对位置信息融入到词向量的点积计算中。

**核心思想**：
不直接生成一个独立的“位置向量”加到词嵌入上，而是**把词嵌入向量旋转一个角度**，旋转的角度由绝对位置 `pos` 决定。这样，当计算两个词的注意力时，它们的点积结果会自然包含相对位置 `pos - pos'` 的信息。

**二维向量的直观例子（方便理解）**：
假设词嵌入是二维向量 `[x, y]`，我们可以把它看作平面直角坐标系里的一个点。对于第 `pos` 个位置，我们将这个向量**逆时针旋转 `θ_pos` 度**：

$$
R(\theta) = \begin{pmatrix}
\cos \theta & -\sin \theta \\
\sin \theta & \cos \theta
\end{pmatrix}
$$

旋转后的向量为：
$$
\begin{pmatrix}
\cos m\theta & -\sin m\theta \\
\sin m\theta & \cos m\theta
\end{pmatrix}
\begin{pmatrix}
x \\
y
\end{pmatrix}
=
\begin{pmatrix}
x \cos m\theta - y \sin m\theta \\
x \sin m\theta + y \cos m\theta
\end{pmatrix}
$$

其中，`m` 就是绝对位置 `pos`，`θ` 的选择和正弦余弦编码类似：`θ_i = 10000^{-2i/d_model}`。

**关键性质**：
当你计算位置 `m` 的旋转向量和位置 `n` 的旋转向量的内积时，结果**只和 `m - n`（相对距离）有关**，和 `m`、`n` 的绝对位置无关。这就完美实现了“用绝对位置的旋转，编码相对位置的信息”。

**高维扩展**：
实际的 `d_model` 通常是 512、768 或 4096。我们把每两维分成一组，每组做一个独立的二维旋转，最后拼接起来，就得到了完整的 RoPE 编码。

**Python 代码实现（简化版）**：
```python
import torch
import math

def precompute_freqs_cis(d_model: int, max_len: int, theta: float = 10000.0):
    # 预计算旋转的频率复数形式 (cos + i sin)
    freqs = 1.0 / (theta ** (torch.arange(0, d_model, 2)[: (d_model // 2)].float() / d_model))
    t = torch.arange(max_len, dtype=freqs.dtype)
    freqs = torch.outer(t, freqs)  # 外积，得到 [max_len, d_model/2]
    # 转换为复数形式：cos + i sin
    freqs_cis = torch.polar(torch.ones_like(freqs), freqs)
    return freqs_cis

def apply_rope(x: torch.Tensor, freqs_cis: torch.Tensor):
    # x 的形状: [Batch, SeqLen, HeadNum, Dim]
    # 把 x 转为复数形式
    x_complex = torch.view_as_complex(x.float().reshape(*x.shape[:-1], -1, 2))
    # 应用旋转 (复数乘法)
    x_rotated = x_complex * freqs_cis[None, :, None, :]
    # 转回实数形式
    x_out = torch.view_as_real(x_rotated).flatten(3)
    return x_out.type_as(x)

# --- 示例使用 ---
d_model = 4  # 小维度演示
max_len = 10
batch_size = 1
seq_len = 2

# 1. 预计算旋转频率
freqs_cis = precompute_freqs_cis(d_model, max_len)

# 2. 模拟一个 Query 向量 (Batch=1, SeqLen=2, Head=1, Dim=4)
q = torch.randn(batch_size, seq_len, 1, d_model)
print("原始 Q:", q)

# 3. 应用 RoPE 旋转
q_rotated = apply_rope(q, freqs_cis)
print("旋转后 Q:", q_rotated)
```

**RoPE 的优势**：
1.  **完美的相对位置编码**：注意力点积天然包含相对距离信息；
2.  **可以外推**：支持超长文本（只要预计算足够长的 `freqs_cis` 即可）；
3.  **计算高效**：通过复数乘法实现，计算开销极低；
4.  **主流标配**：LLaMA 系列、通义千问、百川等几乎所有主流开源大模型都在用。

### 三、总结对比
| 位置嵌入类型 | 生成方式 | 是否可学习 | 能否外推超长序列 | 典型应用模型 |
| :--- | :--- | :--- | :--- | :--- |
| **可学习绝对位置** | 随机初始化矩阵，训练更新 | ✅ 是 | ❌ 不能 | BERT、GPT-2 |
| **正弦余弦绝对位置** | 固定三角函数公式计算 | ❌ 否 | ✅ 能 | Transformer 原论文、T5 |
| **RoPE 旋转位置** | 对词向量进行旋转矩阵变换 | ❌ 否（融入计算） | ✅ 能 | LLaMA 1/2/3、Qwen、Baichuan |
